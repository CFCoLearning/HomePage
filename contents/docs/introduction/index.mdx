在知识爆炸和网络资源丰富的时代，学习已经不再是单向的知识传递，而是一个社会性的过程。

未来的学习方式是否还会像过去和现在这样？拥有一间教室和老师，半年为一个学期，传授体系化的知识？我们认为或许会有更好的解决方案。

**共学，作为一种去中心化的学习实践，为个人成长提供了新的可能。**

## 什么是共学

> 💡 共学没有一个权威定义，它是助人者、行动者领域的热词，强调我们的「共同学习」，是参与者共同选择对话主题、采取行动并对全过程进行反思的实践。共学倡导学习的非个人性，意味着我们需要就获得的信息开展交流与切磋、需要在对话中向彼此的生命经验学习，学习不仅仅是知识的输入，也是与个人成长、社会变革有着积极的交互关系。

在共学中，知识的传递不是单向的，而是通过迭代和反馈的螺旋式成长：**规划 -> 行动 -> 反思 -> 调整 -> 再规划**。

**设计**和**涌现**是推动学习的两个核心机制：
* **设计**提供学习活动的方向感和连续性，使参与者能够清晰地看到学习路径。
* **涌现**则激发了意料之外的合作与创新，促进学习者之间的深入互动。

这种机制使得共学超越了传统标准化课程的框架，成为一个更加开放、灵活的学习过程。角色的边界在共学中被打破，设计者和参与者的身份可以互换，知识的传递通过规划、行动、反思和调整的循环不断深化。

**只有当每位成员都积极参与知识的创建和分享，共学才能最大化地发挥其潜力。**

在我们的共学计划中，希望通过集体的讨论，让每一位成员都能参与到知识的创造和分享中。**这里没有固定的老师和学生，大家都是彼此的伙伴和导师。**

## 发展历程

2024 年 10 月，我在 CFC Studio 例会中发起了 **CFC Co-Leaning & Mini Hackathon 提案**。历时两个月，在一天天的摸鱼中，完成了该文档的 v1.0 版本，标志着 Co-Leaning 提案正式落地，进入试行阶段。

我们的共学计划参考了 **“残酷共学”** 的设计理念，针对 CQUTer 的学习条件，进行了一定的调整和优化（主要将其中的“每周只有两次请假机会”，修改为“每周只有五次请假机会”），最终成为现在的 CFC Co-Learning 计划。我们希望通过这个计划，让每一位 CQUTer 都能参与到知识的创造和分享中，让学习不再是孤独的旅程。


  > 以下内容参考自 https://forum.lxdao.io/t/topic/1654 ，更详细的介绍，请大家自行查看。

  残酷共学是由 Bruce Xu 首创的一种学习模式，目前由 LXDAO 组织并运营残酷共学品牌。共学有很多种，「残酷共学」与之不同的是「残酷」：
  * 你必须每天围绕某个「共学主题」进行学习，每周只有两次请假机会，通常每天至少花费半个小时（最好一个小时）来学习。
  * 你必须提交你的学习证明到对应的 GitHub 仓库来证明你今天学习了。
  * 如果你没有完成上面两点，你会被立刻踢掉并标记为 ❌ 失败。
  * 每期残酷共学以 4 周为一个周期，第一周主要用于共学启动报名和熟悉共学规则，第二周到第四周将正式启动共学，为期 21 天，中途不得加入。
  * 共学方向包括但不限于：英语、以太坊、Web3 技术、DAO 和 加密思潮等，由自主发起。共学的过程包括且不限于：观看视频、阅读书籍与文章、项目实战等。

  报名方式是完全基于 GitHub 程，通过提交 PR 进行申请，合并 PR 之后拥有更新权限。如果你不熟悉 GitHub 和 Git 的操作，请先自行学习。


## 这样的方式好在哪

* **只有三周：** 正好迎合 21 天精通 xxx 的 meme + 21 天养成一个好习惯，不至于太短无法深入学习，也不至于过长很难坚持下来。
* **完全远程：** 整个共学过程都是基于线上进行的，只要有网络就可以学习，没有线下空间，没有地域和时间的限制。
* **开源开放：** 所有学习笔记在 GitHub 上都是开源的，任何人都可以看到所有报名者的打卡记录和学习笔记，即使你没赶上报名，也可以学习大家的资料和笔记。
* **去中心化：** 我们不需要一个老师才可以开始学习，每个人都可以自由的发起一场共学，拉上自己的朋友一起打卡学习。
* **无需许可：** 任何人有想要学习的资料都可以发起共学，任何人也都可以提交报名参与，学习是一个自由的、无需许可的事情。

**学习本身不设置激励，因为学习知识就是最大的奖励。**
